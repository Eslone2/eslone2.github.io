💻 ネットワーク構築 講習会（第二回）

Posted on May 6, 2026 by Eslone2

🎯 今回の最終目標

  • Teratermを使ってネットワーク機器（ルータやスイッチ）に接続できる。
  • 機器の基本的な操作モードの違いを理解し、状態確認コマンドを実行できる。

🧩 1. TCP・UDPとポート番号

第1回で「レイヤ4（トランスポート層）」は信頼性の確保が役割だと言いました。ここではその主役となる2つのプロトコルと、ポート番号について学びます。

TCPとUDPの違い

プロトコル 特徴 主な用途
TCP 信頼性重視。届いたか確認し、無ければ再送する。 Web閲覧、メール、ファイル転送など
UDP スピード重視。確認せず送りっぱなしにする。 動画配信、音声通話、オンラインゲームなど

ポート番号とは？

IPアドレスが「家の住所」だとしたら、ポート番号は「どの部屋（窓口）に届けるか」を指定する番号です。0〜65535番まであり、特に0〜1023番は「ウェルノウンポート」と呼ばれ、主要なサービスに割り当てられています。

ポート サービス名 主な用途
22 SSH 安全なリモート操作（暗号化あり）。サーバー管理の標準。
23 Telnet リモート操作（暗号化なし）。検証用に使われる。
25 / 587 SMTP メールの送信。587番はセキュリティ対応版。
53 DNS 名前解決（URLとIPアドレスの変換）。
67 / 68 DHCP IPアドレスの自動割り当て。
80 HTTP Webサイトの閲覧。
110 / 995 POP3 メールの受信（端末にダウンロードする形式）。
123 NTP 時刻同期。
143 / 993 IMAP メールの受信（サーバー上で管理する形式）。現在の主流。
443 HTTPS セキュアなWebサイトの閲覧。
3306 MySQL データベースへの接続。
3389 RDP Windowsのリモートデスクトップ。

📥 2. Teratermでの接続方法

第1回でインストールした「Teraterm」を使用して、ネットワーク機器（Ciscoルータやスイッチ）に接続します。

多くの場合、初期設定を行う際は「コンソールケーブル」と呼ばれる専用のケーブルを使ってPCと機器を直接繋ぎます（シリアル接続）。

  1. PCとネットワーク機器をコンソールケーブルで接続します。
  2. Teratermを起動し、「シリアル(Serial)」を選択します。
  3. ポート(Port)を選択し、「OK」をクリックします。
注意： シリアルポートの設定（設定 -> シリアルポート）で、スピード(Baud rate)が 9600 になっていることを確認してください（Cisco機器の標準設定）。

⚙️ 2. ネットワーク機器の3つのモード

無事に接続できると、黒い画面に文字が表示されます。Cisco機器（IOS）には、操作の権限によって主に3つのモードが存在します。

① ユーザーEXECモード ( > )

接続直後のモードです。プロンプト（入力待ちの記号）は Router> のようになります。

このモードでは、限られた確認コマンドしか実行できず、設定の変更などは一切できません。

② 特権EXECモード ( # )

すべての確認コマンドや、ファイルの保存・再起動などができるモードです。プロンプトは Router# に変わります。

ユーザーEXECモードから移行するには、以下のコマンドを入力します。

Router> enable
Router#

③ グローバルコンフィギュレーションモード ( (config)# )

機器全体の設定（IPアドレスの付与やルーティングの設定など）を行うモードです。プロンプトは Router(config)# に変わります。

Router# configure terminal
Router(config)#

元のモードに戻るには exit コマンドを使用します。

🔍 3. 状態確認と運用の基本コマンド

特権EXECモード（#）でよく使うコマンドです。Cisco機器と、日本国内でよく使われるAlaxala（アラクサラ）機器のコマンドを比較して見てみましょう。

用途 Cisco (Catalyst等) Alaxala (AXシリーズ)
設定モードへ移行 configure terminal configure
設定の保存 write memory save
動作中の設定確認 show running-config show running-config
保存済み設定の確認 show startup-config show startup-config
機器の再起動 reload reload
バージョン・モデル確認 show version show version
ログの確認 show logging show logging
重要：設定の保存を忘れずに！
設定を変更しただけでは、機器の電源を切ると設定が消えてしまいます。作業が終わったら必ず write memory (Cisco) や save (Alaxala) を実行して、設定を保存しましょう。
