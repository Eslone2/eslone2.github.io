🌐 Mechanisms of IP Addresses, Subnet Masks, and Network Addresses

Mechanism of IP Addresses

An IP address is (roughly speaking) like an address on the internet! Also, IP addresses must never be duplicated, as it would cause communication failures!

For example...

  • Sender: x-x, x-chome, XX-cho, Aomori City, Aomori Prefecture
  • Receiver: x-x, x-chome, xx-cho, Hachinohe City, Aomori Prefecture

If we replace these addresses with IP addresses...

  • Sending device: 172.155.10.1
  • Receiving device: 153.128.11.1

It works like this (though there are minor differences).

*The explanation here will basically use IPv4. We'll explain IPv6 later! (IPv4 operates on 32 bits)

Network Portion and Host Portion

  • The Network Portion is the number that indicates which network it belongs to! It's like the name of an apartment building, e.g., 'XX Hills' (Example: 172.155.10.1).
  • The Host Portion is the number that indicates which specific device it is within the above network! It's like 'Room XXX' in an apartment (Example: 172.155.10.1).

Reference: IP Address - IT Glossary (Japanese)

Mechanism of Subnet Masks

A subnet mask is a manual that explains how to separate the network portion and the host portion.

Reference: Subnet Mask - IT Glossary (Japanese)

For example, suppose the subnet mask is 255.255.255.0. When converted to binary, it becomes 11111111.11111111.11111111.00000000.

  • The part with 11111111.11111111.11111111.00000000 is the Network Portion.
  • The part with 11111111.11111111.11111111.00000000 is the Host Portion.

📝 Practice Problem: Separating Network and Host Portions

Now, let's actually try separating the network and host portions.

  1. When the subnet mask is 255.255.0.0, for IP address 192.168.0.2
  2. When the subnet mask is 11111111.00000000.00000000.00000000 (in binary), for IP address 192.168.163.1

✅ Answers

  1. IP Address: 192.168.0.2 / Subnet Mask: 255.255.0.0
    • Network Portion: 192.168
    • Host Portion: 0.2
  2. IP Address: 192.168.163.1 / Subnet Mask (Decimal): 255.0.0.0
    • Network Portion: 192
    • Host Portion: 168.163.1

📡 Roles of Routers and L2 Switches in a Network

Mechanism of L2 Switches (Layer 2 Switches)

An L2 switch is a device that primarily handles communication within the same network.

  • Role: Delivers data to devices belonging to the same network.
  • Criteria: Makes decisions by looking at the MAC address of the destination device.

Mechanism of Routers (Layer 3 Switches/Routers)

A router is a device that primarily handles communication between different networks.

  • Role: Relays data between different networks and sends it along the optimal path.
  • Criteria: Makes decisions by looking at the IP address of the destination device.
Device Type Layer Processed Decision Criteria Main Role
L2 Switch Layer 2 MAC Address Communication within the same network
Router Layer 3 IP Address Communication between different networks
