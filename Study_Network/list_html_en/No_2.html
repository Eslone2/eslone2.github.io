🧩 1. TCP, UDP, and Port Numbers

In Session 1, we mentioned that "Layer 4 (Transport Layer)" is responsible for ensuring reliability. Here, we will learn about the two main protocols and port numbers.

Differences Between TCP and UDP

Protocol Characteristics Main Uses
TCP Reliability focused. Verifies if data arrived and resends if missing. Web browsing, Email, File transfer, etc.
UDP Speed focused. Sends data without waiting for confirmation. Video streaming, VoIP, Online gaming, etc.

What are Port Numbers?

If an IP address is like a "house address," a port number is like specifying "which room (window) to deliver to." They range from 0 to 65535, with 0 to 1023 being "Well-known Ports" reserved for major services.

Port Service Name Main Roles / Keywords
22 SSH Secure remote operation (encrypted). Standard for server management.
23 Telnet Remote operation (unencrypted). Used for testing/verification.
25 / 587 SMTP Sending emails. Port 587 is the secure version.
53 DNS Name resolution (converting URLs to IP addresses).
67 / 68 DHCP Automatic IP address assignment.
80 HTTP Web browsing.
110 / 995 POP3 Receiving emails (downloading to devices).
123 NTP Time synchronization.
143 / 993 IMAP Receiving emails (managed on the server side). Modern standard.
443 HTTPS Secure web browsing.
3306 MySQL Connecting to databases.
3389 RDP Windows Remote Desktop.

📥 2. How to Connect with Teraterm

Use "Teraterm" installed in Session 1 to connect to network devices (Cisco routers and switches).

In most cases, when performing initial configuration, you connect the PC directly to the device using a special cable called a "console cable" (serial connection).

  1. Connect the PC and the network device with a console cable.
  2. Launch Teraterm and select "Serial".
  3. Select the Port and click "OK".
Note: Ensure that the speed (Baud rate) is set to 9600 in the serial port settings (Setup -> Serial port) (the standard setting for Cisco devices).

⚙️ 2. Three Modes of Network Devices

Once successfully connected, text will appear on a black screen. Cisco devices (IOS) mainly have three modes depending on operating privileges.

① User EXEC Mode ( > )

This is the mode immediately after connecting. The prompt (input waiting symbol) looks like Router>.

In this mode, you can only execute limited verification commands, and no configuration changes can be made.

② Privileged EXEC Mode ( # )

In this mode, you can execute all verification commands, save files, restart, etc. The prompt changes to Router#.

To transition from User EXEC mode, enter the following command.

Router> enable
Router#

③ Global Configuration Mode ( (config)# )

This is the mode for configuring the entire device (such as assigning IP addresses and setting up routing). The prompt changes to Router(config)#.

Router# configure terminal
Router(config)#

Use the exit command to return to the previous mode.

🔍 3. Basic Commands for Status and Operations

These are commands frequently used in Privileged EXEC Mode (#). Let's compare commands for Cisco devices with Alaxala devices, which are common in Japan.

Purpose Cisco (Catalyst, etc.) Alaxala (AX Series)
Enter Config Mode configure terminal configure
Save Configuration write memory save
Check Running Config show running-config show running-config
Check Startup Config show startup-config show startup-config
Restart Device reload reload
Check Version/Model show version show version
Check Logs show logging show logging
Important: Don't forget to save your settings!
Simply changing the settings is not enough; they will disappear if the device is powered off. After you finish your work, always run write memory (Cisco) or save (Alaxala) to save your configuration.
