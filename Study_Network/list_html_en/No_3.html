💻 Network Construction Workshop (Session 3)

Posted on May 6, 2026 by Eslone | Workshop Materials

🎯 Final Goals for this Session

  • Understand wireless LAN standards and the differences between an Access Point (AP) and a Router (RT).
  • Understand the mechanisms of "encryption (public key / private key)", which are essential for secure communication.

📶 1. What is the Wireless LAN Standard "IEEE802.11"?

When connecting smartphones or PCs to Wi-Fi, a communication standard (rule) called "IEEE802.11" is used behind the scenes.

Differences in Frequency Bands (2.4GHz / 5GHz / 6GHz)

Wi-Fi primarily uses three frequency bands (paths for radio waves).

  • 2.4GHz Band: Strong against obstacles (like walls) but prone to interference from microwaves or Bluetooth, and relatively slower.
  • 5GHz Band: Fast and less prone to interference from other appliances, but weak against obstacles and has a shorter range.
  • 6GHz Band: A newly added band. Less crowded and free from interference, enabling ultra-fast and stable communication. However, it is the weakest against obstacles.

Evolution of Representative Standards

As technology advances, the letter at the end of the standard changes. Recently, it has become more common to refer to them simply by numbers like "Wi-Fi X".

Standard Name Simple Name Features & Frequency Band
IEEE 802.11n Wi-Fi 4 Supports both 2.4GHz / 5GHz. The previous standard.
IEEE 802.11ac Wi-Fi 5 5GHz only. Communication speed dramatically improved.
IEEE 802.11ax Wi-Fi 6 Supports both 2.4GHz / 5GHz. Very strong for simultaneous connections of multiple devices. The current mainstream.
IEEE 802.11ax Wi-Fi 6E Extended version of Wi-Fi 6. Now supports the 6GHz band, enabling even more stable and ultra-fast communication.
IEEE 802.11be Wi-Fi 7 Supports 2.4GHz / 5GHz / 6GHz. The next-generation highest speed standard, capable of bundling multiple frequency bands simultaneously (MLO).

🔄 2. Differences Between Routers (RT) and Access Points (AP)

In networking, "Routers" and "Access Points" are often confused. Let's clarify their true roles.

🌐 Router (Router / RT)

Role: A guide that connects different networks (Layer 3)

It is a device to connect the "internet world" and the "network inside the house (LAN)". It also has functions like distributing IP addresses to connected devices (DHCP). Originally, it does not have the function to transmit wireless (Wi-Fi) signals.

📡 Access Point (Access Point / AP)

Role: A "bridge" that allows wireless devices to join the network (Layer 2)

It is a device that converts wired LAN cables into wireless signals and allows smartphones and PCs to join the network. It's essentially a "wireless switch (hub)". It does not have the function to distribute IP addresses or connect to the internet.

💡 The Truth About Household "Wi-Fi Routers":
The "Wi-Fi router" in your house is actually an extremely convenient multi-function device combining [Router Function] + [AP Function] + [Switch Function] in one box. In companies, it is common to install routers and APs as separate devices.

🔒 3. Basics of Encryption: To Send Information Securely

On the internet, data is delivered via various devices like a bucket brigade. If sent as it is (plaintext), there is a risk of it being stolen and read along the way, so "encryption" is essential.

Common Key Cryptography

A method where the "key to lock" and the "key to unlock" are the same. Processing speed is fast, but there is a major problem: "How do you securely deliver that key to someone far away?"

Public Key Cryptography (Important!)

Public Key Cryptography solved this problem. Here, two paired keys, a "Public Key" and a "Private Key", are used.

  • Public Key (A padlock anyone can use): A key solely for locking data, safe to give to anyone.
  • Private Key (A key only you have): The only key in the world that can unlock the lock. Must never be given to anyone.

【Mechanism】
If Person A wants to send a secret message to Person B, Person A locks the message with "Person B's Public Key (padlock)" distributed by Person B, and sends it. Because this padlock can only be opened with "Person B's Private Key" that Person B holds, even if someone steals it along the way, they cannot see the contents!

🔑 4. Familiar Examples of Encryption

① Browsing Websites (HTTPS)
Sites whose URLs start with https:// and have a "padlock icon" in the browser have their communication encrypted. Thanks to this technology, sending credit card information is safe.

② Connecting to Network Devices (SSH)
In Session 2, we used "Teraterm" to connect to network devices. A frequently used connection method here is SSH (Secure Shell). This also combines "Public Key Cryptography", etc., encrypting passwords and operation commands so you can securely operate the device.
