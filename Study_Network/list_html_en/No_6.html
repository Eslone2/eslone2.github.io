💻 Network Construction Workshop (Session 6)

Posted on May 12, 2026 by Eslone2

🎯 Final Goals for this Session

  • Understand mechanisms for server load balancing and high availability.
  • Understand the differences between infrastructure foundations like on-premises, virtualization, and containers.
  • Understand services that support network operations, such as DHCP and Active Directory.

⚖️ Server Load Balancing and Availability

Technologies designed to keep services running even under heavy traffic or hardware failure.

Load Balancer

A device that distributes incoming traffic from a single "entry point" to multiple backend servers. This ensures service continuity even if one server fails (improving availability).

CDN (Content Delivery Network)

A system that delivers content via globally distributed "cache servers." By delivering heavy data like images or videos from locations close to users, it achieves both faster speeds and reduced load on the main server.

🐒 Chaos Engineering and Resilience

In today's world of massive and complex systems, it is essential to design with the assumption that "things will eventually break."

Chaos Engineering

An approach where you intentionally trigger failures—such as "shutting down servers" or "injecting network latency"—in a production environment to test if the system can withstand them.

💡 Chaos Monkey

A famous tool developed by Netflix that randomly terminates server instances in the cloud. It forces engineers to design systems that can survive unpredictable failures.

Resilience

The ability of a system to automatically recover and continue service even when a failure occurs. Related technologies include Auto-scaling (adjusting server counts based on load) and Circuit Breakers (isolating failed components to prevent cascading failures).

🏗️ Infrastructure Foundation Models

The different "platforms" on which systems are run.

🏢 On-premises

The traditional model where a company purchases physical servers and installs/operates them in their own facilities (such as a data center).

💻 Virtualization (VM) and Containers (Docker)

  • Virtualization (Virtual Machine): Technology to run multiple independent operating systems on a single physical server (e.g., VMware ESXi).
  • Container (Docker): Technology to logically isolate OS resources to create lightweight and fast-starting execution environments.

🛠️ Services Supporting Network Operations

DHCP (Dynamic Host Configuration Protocol)

A mechanism that automatically assigns IP addresses and other settings to PCs and smartphones. It saves the effort of manual configuration and prevents address duplication.

Active Directory (AD)

A directory service primarily for Windows-based corporate networks used to centrally manage user information, PCs, and permissions. It works in close coordination with DNS.

📝 Summary

  • Load Balancer: Distributes access among multiple servers.
  • CDN: Uses caches for high-speed, low-load content delivery.
  • Chaos Engineering: A method of finding weaknesses by intentionally causing failures.
  • Resilience: The "recovery power" to automatically recover from failures.
  • Virtualization/Containers: Technologies for efficient physical server utilization.
  • DHCP: Automatically distributes IP addresses.
  • Active Directory: Centrally manages users and PCs.
